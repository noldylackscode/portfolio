Professional Experience Project – 🤖 Robotic Process Automation – Download And Upload Process

Images are not available for this project due to the confidential nature of the RPA software used.

🏢 Company: Nidec Philippines Corporation

🗺️ Location: 136 North Science Avenue Extension, Laguna Technopark SEZ, Binan, Laguna, Philippines

🗓️ Duration: October 21, 2025 – April 20, 2026

🛠️ Tools & Technologies: (Confidential) RPA software, Excel, SQL, SAP, Warehouse Management Systems; C# & Visual Studio (for testing in the test environment); Remote Desktop & VNCViewer (for live environment management)

📌 Project Overview

Implemented an automated workflow for end-to-end SAP and WMS data handling. Key responsibilities included:

  • Downloading batch data from the Warehouse Management System (WMS) for processing.
  • Uploading processed data into SAP, ensuring accurate transfers and preventing double postings caused by temporary network spikes.
  • Logging success/error messages for each batch, storing backups, and maintaining comparison records for future uploads.
  • Automating workflow control: disabling the process via .bat script on errors, testing fixes in the test environment with C# & Visual Studio, and resuming in the live environment using Remote Desktop & VNCViewer. Implemented an auto-start feature to restart the workflow automatically once the User Interface is stable.
  • Leveraging image detection in RPA Software to accurately perform mouse clicks and keyboard events on the SAP and WMS interfaces, ensuring reliable interaction with the user interface even under dynamic screen conditions.

Operational Details: Runs continuously 24/7 in 15-minute cycles with a 4-minute pause between batches to maintain data integrity across both databases.

Impact: Reduced monthly errors from over 1,000 to ~70 per audit, ensured reliable data transfer between WMS and SAP, minimized manual intervention, and improved audit accuracy. Fully eliminated manual handling, allowing the team to focus on higher-value tasks and boosting operational efficiency.

Professional Experience Project – 🤖 Robotic Process Automation – Yield Loss Data Download & Auto-Summarization

Images are not available for this project due to the confidential nature of the RPA software used.

🏢 Company: Nidec Philippines Corporation

🗺️ Location: 136 North Science Avenue Extension, Laguna Technopark SEZ, Binan, Laguna, Philippines

🗓️ Duration: October 21, 2025 – April 20, 2026

🛠️ Tools & Technologies: (Confidential) RPA Software, Excel VBA (Macro Automation), Excel Data Processing, Image Recognition Automation for Mouse & Keyboard Interaction

📌 Project Overview

Developed an automated RPA workflow designed to download, process, and summarize Yield Loss data used for operational analysis. The system handles multiple model classifications and part-based datasets, ensuring accurate and consistent daily reporting while eliminating manual data preparation.

  • Implemented an RPA process responsible for automatically downloading 114 different yield loss datasets across multiple model classifications and parts.
  • Utilized image recognition automation to accurately control mouse movements and keyboard inputs within the system interface, ensuring reliable navigation and file downloads.
  • Designed and developed a custom Excel VBA macro that automatically reads all downloaded files from a designated folder using the company’s predefined naming structure.
  • The macro extracts the required data from the previous day’s Yield Loss reports and compiles them into a centralized summarized Excel report.
  • Implemented an automated Save Copy As workflow to generate multiple backup copies for redundancy and data traceability.
  • Automated file organization by transferring:
    • Raw downloaded files to an internal local storage folder for internal reference.
    • Summarized reports to designated backup locations for the IT Department.
    • Final processed files to a shared location accessible by the client for operational use.

Operational Details: The automation runs daily, retrieving the previous day's Yield Loss data and executing a fully automated pipeline consisting of file downloading, macro-based summarization, backup creation, and structured file distribution.

Impact: Eliminated manual downloading and data consolidation tasks across 114 datasets, significantly improving processing speed, data accuracy, and reporting consistency while providing reliable daily summarized data for both internal teams and external clients.

🎓 Thesis Project – Student Intervention Monitoring System

🏫 School: Trimex Colleges

👨‍💻 Role: Co-Developer (Backend & Frontend)

🛠️ Technologies: Visual Basic, MySQL, PHPMailer, RFID System

📌 Project Overview

Developed in collaboration with a teammate, this system automates student monitoring using an RFID-based attendance and grading system. Students who are absent for three consecutive days or have an average below 75 in major subjects are flagged as "At Risk," labeled as "Subject for Improvement" in our research documentation.

We built the system entirely using VB.NET for both the frontend and backend, with MySQL as the database. Email alerts are automatically sent via PHPMailer when specific risk conditions are met. The system also tracks student intervention statuses such as "At Risk," "Subject for Improvement," "Failed," or "Intervention Completed."

Key features include teacher-subject management, grade input, student performance tracking, searchable records, and full CRUD functionality.
